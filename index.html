Springbok JS demo

Springbok binds a module to the specified HTML element. Example:

<div data-module="myNamespace.myWidget"></div>

Or, alternatively in AMD style:

<div data-module="modules/myWidget"></div>

Demo

<div data-module="modules/WidgetA"></div>

Widget A initial content

<div data-module="MyNamespace.WidgetB,WidgetD"></div>

Widget B & D initial content

Please inspect the resources to see how (simple) it works, and how to set it up your way.

Details (traditional vs. AMD-style)

Widget definitions

  • Widget A & B are straight-forward modules (traditional JS objects).
  • Widget C is an AMD module.
  • Widget D is a hybrid module (AMD, CommonJS, traditional).

The setup

  • Using traditional script loading, widget styles A+B and D work (C doesn't).
  • Using any AMD compliant script/module loader, widget C and D work (A+B doesn't).
  • D works in both cases, but needs some extra "ceremony" code.
  • Last but not least, you can also use your own script loader (example: DuckJS)
